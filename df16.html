3 Introduction to Git-Based Continuous Delivery with Heroku Pipelines

3.1 Overview

  1. Description

    Continuous Delivery is the prescription for code quality, product agility, and engineering velocity. 継続的デリバリはコード品質、プロダクトのアジリティ、開発の速度に寄与する手法や。

    In this session, you'll learn how Heroku Flow makes Continuous Delivery simple and flexible. sこのセッションでは、*Heroku Flow* がCDをごくシンプルに柔軟にしてくれるってところを見せたる。

    The Git-based methodology provides for a Pipeline through which code begins as a fully executable pull request, is automatically merged to test and staging apps, and is then deployed to production with a single click. Gitベースの方法論によって提供される Pipeline機能っちゅうやつは、コードが実行可能なプルリクを起点にして、自動的にテスト・ステージングアプリ環境にマージして(preview appsのこと？)、 そして本番環境へのデプロイまでをワンポチで行えるんや。

    Some customers are able to deploy features and fixes up to 500+ times a week, with very few rollbacks from production. 新機能や修正のデプロイを週に500以上もやって、本番環境のごくごく稀なロールバックもやる、そんなことができるお客さんもおるんや。
